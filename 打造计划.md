# 个人微信 Agent 打造计划

当前唯一执行记录见 `计划书.md`。本文件只保留长期不变量，避免历史阶段说明再次成为错误的当前事实。

## 长期不变量

1. 真实发送默认关闭，只有显式配置与完整投递守卫同时满足时才允许进入发送链。
2. `data/send_bridge` 是投递证据链，历史清理、通道清理和一般诊断清理均不得删除或重放其中内容。
3. 对话通道、账本、session、确认队列、任务和侧栏运行态以 SQLite 为单向权威；JSON/JSONL/Markdown 仅作可读投影，不反向导入。
4. 历史清理后从空对话权威库启动，不迁移旧通道、旧账本、旧 session 或旧侧栏运行态。
5. 页面 OCR 与窗口快照不得触发回复；不可信快照只能为已存在的可信通道补充只读上下文，未知通道直接阻断。
6. 同一会话内消息线性处理；历史回填不得重置 session、触发主动回复或消耗群聊冷却。
7. 前端破坏性操作必须确认；同作用域命令串行化，刷新、保存与关闭面板之间不得发生陈旧状态覆盖。

## 发布门槛

- `audit-plan` 的 `plan_residual_count` 必须为 `0`。
- 全量 Python 测试、Python 编译、JavaScript 语法检查和 `git diff --check` 全部通过。
- 桌面与窄屏浏览器交互验收通过，无重叠、无陈旧重复命令、无保存覆盖。
- `send_enabled` 保持 `false`，`send_bridge` 证据文件哈希与清理前一致。
- 改动提交到功能分支，合并 `main` 后推送 GitHub。
