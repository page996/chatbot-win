# 实机稳定性核验记录

更新时间：2026-06-30

## 当前结论

1. 微信窗口发现与 guarded 发送探测仍可用。
   - 真实发送必须继续依赖 `windows_guarded` 的前台窗口守卫。
   - 发送前仍应通过确认队列或自动发送开关审计。
2. 页面 OCR 输入链路已废弃。
   - `ocr-snapshot`、`poll-ocr-window`、`ocr-window-diagnose` 只返回 `deprecated`。
   - 这些命令不会截图、不会 OCR、不会写 ledger、不会入发送队列。
3. OCR 只保留为文件层工具。
   - 图片、扫描件、文档内图片等素材由文件中间层和 `vision.ocr`/`ocr-image` 调用。
   - 不再把微信页面截图 OCR 成对话快照。
4. 页面/窗口抓取只保留纯抓取。
   - `wechat-capture` 可以用于手工取证、窗口验证或后续后台抓取调试。
   - 抓到的图片不会自动进入对话上下文。

## 输入路径边界

稳定运行优先级：

1. 后端事件源：`poll-backend-events`。
2. 文件 watcher：`scan-backend-files` 产生事件后交给后端事件源。
3. 文本快照：`poll-snapshot`，仅用于明确格式的文本快照。
4. 剪贴板/UIA：`poll-clipboard`、`wechat-snapshot`，只作为只读调试或过渡输入。
5. 纯窗口抓取：`wechat-capture`，只取图，不 OCR，不写 ledger。

禁止路径：

- 不使用 OCR 阅读微信页面。
- 不使用 OCR 截图推断消息、文件卡片或语音转文字。
- 不把页面 OCR 结果写入 `conversation_ledgers`。
