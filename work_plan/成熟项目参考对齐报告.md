# 成熟项目参考对齐报告

更新时间：2026-06-30

## 参考项目与文档

本项目继续参考成熟 agent / memory 架构，但不直接引入它们作为运行时依赖：

1. LangGraph / LangChain Memory
   - 参考点：短期记忆按 thread/session 隔离，长期记忆按 namespace/key 存储。
2. OpenAI Agents SDK Sessions
   - 参考点：session 负责跨多轮 agent run 维护对话历史，应用侧拥有状态、审批和工具编排边界。
3. LlamaIndex Memory
   - 参考点：短期聊天历史受 token limit 约束，超限后 flush/summarize 到长期记忆。
4. Letta / MemGPT Memory
   - 参考点：总是注入上下文的 core memory 与按需检索的 archival memory 分层。

## OCR 边界

页面 OCR 输入已废弃：

- 不再通过 OCR 阅读微信页面。
- 不再把截图 OCR 结果转换成微信消息快照。
- 不再把页面 OCR 结果写入 ledger。

OCR 只保留为工具层能力：

- 文件中间层处理图片、扫描件、文档内图片时可以调用 OCR。
- `ocr-image` 与 `vision.ocr` 仍用于文件/素材解析。
- `wechat-capture` 只做纯窗口抓取，不做 OCR，不写 ledger。

## 后续靠拢项

1. 长期记忆仍是确定性抽取，不是成熟项目中的 LLM-assisted memory extraction。
   - 后续可给 `MemoryMaintainer` 增加可插拔 summarizer/extractor。
2. 归档记忆还没有向量或语义检索。
   - 后续可为 ledger text block、文件解析块、网页注释建立 SQLite FTS 或向量索引。
3. 文件、网页、语音处理链仍需更完整的 artifact manifest。
   - 工具结果应统一产出 manifest，再由 maintainer/assembler 引用。
4. UI 抓取不是生产级消息源。
   - 后续应接真实后台消息事件源；UI 仅保留纯窗口抓取诊断；OCR 只保留为文件层工具。

## 当前推荐架构

继续保持“ledger 是唯一对话事实源，memory 是 ledger 派生物”的结构，不恢复旧 `context_store` 消息历史。

上下文进入模型的顺序：

1. 当前 conversation + current session 的 active ledger entries。
2. 当前消息显式 quote 命中的跨 session 邻近窗口。
3. 当前 session 的 core memory：summary/preferences/entities。
4. 相关文件、网页、附件 artifact refs。
5. 在 token budget 内保留最近消息；超出预算先裁剪 recent，不裁剪 quote/files/memory。
